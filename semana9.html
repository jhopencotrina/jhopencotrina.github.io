Semana 9: Next JS CSR / SSR

DEFINICIONES

Next.js es un framework de desarrollo web React que simplifica y agiliza la creación de aplicaciones web.

Proporciona un conjunto de utilidades y convenciones para la creación de rutas, gestión del estado y la posibilidad de renderizar páginas tanto en el lado del servidor como en el lado del cliente.

Dentro de su uso y utilidad de estan:

1. Interactividad Rápida: Permite una carga inicial rápida, ya que solo se envía el esqueleto de la aplicación al cliente, y luego se realiza la renderización en el navegador.

2. Aplicaciones de Una Página (SPA): Adecuado para aplicaciones que requieren una interactividad intensiva en el cliente y funcionan como SPAs, donde las transiciones entre páginas se gestionan sin recargar la página completa.

PROCEDIMIENTOS

Next.js utiliza una serie de procedimientos y conceptos para su ejecución.

Aquí una visión de los procedimientos clave que están involucrados en el funcionamiento de una aplicación que utiliza Next.js:

Se debe configurar un proyecto Next.js utilizando la herramienta de línea de comandos create-next-app

Luego se ejecuta el comando: npm run dev

Despues Next.js utiliza un sistema de enrutamiento basado en archivos. Los archivos en la carpeta pages determinan las rutas de la aplicación.

Para la creación de paginas dinamicas y estaticas utiliza las funciones especiales como getStaticProps y getServerSideProps para controlar la obtención de datos.

Se puedes gestionar los estilos de los componentes utilizando módulos CSS, Styled JSX, o integrando con bibliotecas populares como SCSS o Tailwind CSS

Next.js permite la creación de API Routes para gestionar solicitudes y respuestas en el servidor. Estos archivos se encuentran en la carpeta pages/api.

Para desplegar la aplicación, se puede utilizar diferentes servicios, como Vercel, que tiene una integración estrecha con Next.js. También se puede desplegar la aplicación en otros servicios como AWS, Netlify, entre otros.

RESULTADOS

El uso de Next.js puede proporcionar una experiencia de desarrollo más eficiente, mejor rendimiento, y facilitar la implementación de estrategias avanzadas, como la representación del lado del servidor y el SEO mejorado.

La configuración inicial de un proyecto Next.js es rápida y sencilla utilizando create-next-app o configurando manualmente. Además, la recarga en tiempo real y la actualización automática durante el desarrollo local facilitan el desarrollo rápido y eficiente.

Se puedes crear API Routes en la carpeta pages/api para gestionar solicitudes y respuestas en el servidor. Esto simplifica la creación de puntos de conexión de API en la aplicación.
